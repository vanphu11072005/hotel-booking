# ========================================================================================================================================
# =============================================================== `migrate` ==============================================================
# ========================================================================================================================================
1. `use strict` - Kích hoạt chế độ nghiêm ngặt (strict mode).
        - Mục đích:
            + Nó giúp phát hiện lỗi tiềm ẩn và bảo vệ code
            + Nó khiến JavaScript "nghiêm khắc hơn" với lập trình viên

2. `/** @type {import('sequelize-cli').Migration} */` - JSDOC type annotation (chú thích kiểu dữ liệu cho JavaScript)
        - Giúp IDE (như VS Code) hiểu kiểu dữ liệu của biến hoặc module.
    - "File này export ra một object có kiểu là Migration đc định nghĩa trong packge sequelize-cli

3. `queryInterface` - Là đối tượng trung gian (interface) giúp tương tác trực tiếp với database mà không cần viết SQL thủ công.

4. `Sequelize` - Là đối tượng Sequelize DataTypes (kiểu dữ liệu) được truyền sẵn vào hàm.

# ========================================================================================================================================
# =============================================================== `model` ================================================================
# ========================================================================================================================================
1. `const { Model } = require('sequelize')` - Lấy lớp Model từ thư viện Seuqelize để tạo các bảng (models) trong cơ sở dữ liệu.

2. `DataTypes` - Dùng để định nghĩa kiểu dữ liệu cho các cột trong bảng.

3. `underscored: true` - Là tất cả các cột sẽ dùng snake_case thay vì camelCase.
